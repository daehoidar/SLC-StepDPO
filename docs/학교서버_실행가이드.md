# 학교 서버(gate1) 본 학습 실행 가이드

RunPod에서 검증 완료한 파이프라인을 학교 GPU 서버에서 풀스케일로 돌린다.
판정 = **persona/math 모두 gpt-4o-mini + 정규식보강 + few-shot**, 최종 평가 = **gpt-4o**.

## 0. 사전 점검 — ⚠️ 가장 먼저 확인
**GPU compute 노드에서 외부 인터넷(api.openai.com)이 되는가?**
판정 단계가 GPU 노드에서 OpenAI API를 호출한다. 안 되면 단일 잡이 실패한다.
```bash
# 로그인 노드에서 GPU 잡으로 테스트
srun -p gpu6 --gres=gpu:1 --pty bash -c 'curl -sI https://api.openai.com/v1/models | head -1'
```
- 200/401 응답 → 인터넷 됨 → **경로 A**(단일 sbatch)
- timeout/실패 → 인터넷 안 됨 → **경로 B**(분리 실행, 아래) 필요 — 알려주면 분리 스크립트 제공

## 2. 환경
```bash
ssh gate1
conda activate persona-dpo
pip install scipy matplotlib            # 통계/플롯 (없으면)
pip install vllm                        # 샘플링 가속 (권장; 실패해도 transformers fallback)

# 판정용 키 — 팀 키 먼저, 소진(insufficient_quota)되면 자동으로 본인 키로 failover
export OPENAI_API_KEY=sk-팀키
export OPENAI_API_KEY_FALLBACK=sk-본인키     # 선택: 여러 개면 쉼표로 구분
```
> 키 failover는 `openai_client.make_openai_client()`가 처리한다. 1순위 키가 **할당량
> 소진**되면 다음 키로 자동 전환(로그에 `[openai-failover]` 출력). 일시적 rate-limit은
> 전환 안 하고 기존 재시도에 맡긴다. `OPENAI_API_KEY` 하나만 주면 기존과 동일 동작.

## 3. 풀런 (경로 A — compute 노드 인터넷 OK)
한 방에 SFT→merge→샘플링→judge→페어→학습:
```bash
cd ~/project/Persona-Step-DPO
export OPENAI_API_KEY=sk-...
MAX_ROWS=0 K_SAMPLES=8 \
  CONFIG=configs/default.yaml OUT=checkpoints/bc_stepdpo \
  sbatch scripts/bc_stepdpo_pipeline_slurm.sh
# MAX_ROWS=0 = 전체 300행(50문제×6), K=8
# judge 기본값이 이미 mini → 따로 안 줘도 됨
tail -f "$(ls -t logs/bc_pipe_*.out | head -1)"
```
> SFT 어댑터가 없으면 파이프라인이 자동으로 SFT부터. 이미 있으면 스킵.
> ⚠️ SFT는 [sft_slurm.sh](../scripts/sft_slurm.sh)로 먼저 돌려 `checkpoints/sft_qwen3_1.7b_eos`를
> 만들어두면 파이프라인에서 `ADAPTER=checkpoints/sft_qwen3_1.7b_eos`로 지정 가능.

예상: 비용 ~$5(mini), 시간 = 샘플링(vLLM면 ~1h / transformers면 ~8h) + 학습.

## 4. 결과
- `data_pipeline/output/preference_pairs.jsonl` — 선호쌍 (Type-1/Type-2)
- `data_pipeline/output/samples_with_persona_labels.jsonl` — 샘플+persona 라벨
- `docs/BC-StepDPO_스모크결과.md` — 정리 문서
- `checkpoints/bc_stepdpo/` — 학습된 모델

## 5. 최종 평가 (gpt-4o)
```bash
python data_pipeline/5_evaluate.py \
    --model checkpoints/bc_stepdpo \
    --test-set <held-out 테스트셋> \
    --personas-path personas.json \
    --output eval_results.json
# 5_evaluate는 gpt-4o 하드코딩 (메모리: 최종 평가=4o)
```
> ⚠️ 테스트셋은 데이터 구축(seed 50문제)에 **안 쓴 held-out 문제**여야 함 (누수 방지).

## 6. ablation
`configs/default.yaml`의 toggle로 12-run 그리드 (vanilla DPO / Step-DPO / BC-StepDPO 등).
데이터(preference_pairs)는 1회 구축 후 재사용, 모델만 다시 학습.

## 핵심 수정 사항 (RunPod 검증 중 발견·반영)

- `2_train_sft.py`: EOS 부착 (반복 생성 방지)
- `configs/default.yaml`: BC-StepDPO batch_size 4→2 (OOM 방지)
